# 從 0 建立：Playwright Real-Time AI Investment Research Agent

請從零開發一個「即時 AI 投資研究工具」。

第一版專注於長榮海運 2603.TW，但架構要能擴充到其他台股、美股、ETF。

此工具不是自動下單工具，也不是自動通知工具。

使用方式：

使用者輸入股票代號，按下「立即分析」，系統才即時抓取資料、搜尋資料、交給 AI 分析，並產出報告。

---

# 1. 核心目標

建立一個工具，能做到：

1. 使用者輸入股票代號，例如 2603.TW
2. 使用者選擇 General Mode 或 Personalized Mode
3. 按下「立即分析」
4. 後端開始抓取資料
5. 若 API 資料不足，啟動 Playwright/Chromium 網頁爬蟲
6. 若爬蟲仍不足，啟動 Web Search Fallback
7. 將所有資料整理成結構化 JSON
8. 交給 OpenAI API 分析
9. 產出 Markdown 報告
10. 前端顯示報告、資料來源、資料缺漏與信心分數

---

# 2. 重要設計原則

## 必須做到

* 資料優先，AI 不可幻想
* API 抓不到，不要直接 Data Missing，要先嘗試 Playwright 爬蟲
* Playwright 爬不到，再啟動 Web Search Fallback
* 搜尋結果要交給 AI 萃取成結構化 JSON
* 報告必須區分：

  * exact data
  * scraped data
  * search-inferred data
  * missing data
* 每筆資料保留來源 URL
* 資料過舊要標示 stale
* 不要自動下單
* 不要自動通知
* 使用者按下分析才執行

## 禁止

* 不可硬塞假資料
* 不可抓不到 SCFI 卻假設運價強
* 不可只用新聞摘要代替分析
* 不可只根據技術線做結論
* 不可在資料不足時給 Strong Bullish / Strong Bearish
* 不可忽略資料來源

# 4. 分析模式

## General Mode

只分析市場與股票本身，不讀取使用者持股資料。

輸出：

* 今日市場狀態
* 短線趨勢
* 中線趨勢
* 支撐區
* 壓力區
* 是否過熱
* 多方理由
* 空方理由
* 風險
* 資料缺漏

---

## Personalized Mode

讀取 `user_profile.yaml`，代入使用者持股、成本、稅率、風險偏好，給出個人化建議。

預設設定：

```yaml
user:
  tax_rate: 0.20
  investment_style: value_investing
  risk_tolerance: medium
  dividend_tax_sensitive: true
  avoid_full_exit: true

positions:
  - symbol: "2603.TW"
    name: "長榮海運"
    lots: 30
    shares: 30000
    average_cost: 192
    core_lots: 20
    flexible_lots: 10

strategy:
  target_sell_zones:
    - price_range: [245, 250]
      action: "consider_sell"
      suggested_lots: [2, 3]
    - price_range: [255, 260]
      action: "consider_sell"
      suggested_lots: [3, 5]
    - price_range: [270, 280]
      action: "re_evaluate"
      suggested_lots: [3, 5]
    - price_range: [290, 300]
      action: "take_profit_or_re_evaluate"
      suggested_lots: [3, 5]

  buyback_zones:
    - price_range: [220, 230]
      action: "consider_buyback"
    - price_range: [200, 220]
      action: "strong_buyback_watch"
    - price_range: [180, 200]
      action: "aggressive_buyback_if_fundamentals_not_broken"
```

# 5. 資料取得順序

所有資料模組都採用以下順序：

```text
Primary API
↓
Playwright / Chromium Crawler
↓
Web Search Fallback
↓
AI Extraction
↓
Data Missing
```

回傳資料必須包含：

```json
{
  "value": null,
  "status": "ok / scraped / inferred_from_search / missing / stale",
  "confidence": 0.0,
  "source": "",
  "source_url": "",
  "as_of": "",
  "notes": ""
}
```

## Search Agent 規則

* 至少交叉比對 2~3 個來源
* 優先使用最近 7 天資料
* 超過 14 天標示 stale
* 不可憑空補數字
* 找不到精確數字時，可以回傳 trend，但 confidence 必須降低
* 所有搜尋結果保留 URL
* 快速搜尋模式總時間控制在 10 秒內
* 30 分鐘內相同 query 使用 SQLite cache
* 若搜尋結果互相矛盾，標示 conflict

```python
class PlaywrightCrawler:
    async def fetch_page(self, url: str) -> str:
        ...
    async def extract_text(self, url: str) -> str:
        ...
    async def extract_tables(self, url: str) -> list:
        ...
    async def screenshot_on_error(self, url: str) -> str:
        ...
```

要求：

* headless Chromium
* 設定 timeout
* 設定 user-agent
* 每站限速
* 錯誤時保留 screenshot
* 不繞過登入
* 不破解 CAPTCHA
* 尊重 robots.txt 與網站條款

# 10. 資料完整度規則

對長榮 2603，資料權重：

```text
SCFI / Freight：35%
法人籌碼：25%
基本面：15%
技術面：15%
新聞 / 紅海：10%
```

規則：

* 若缺少 SCFI，不可輸出 Strong Bullish
* 若缺少 SCFI + News，最高只能 Neutral 或 Insufficient Data
* 若只有技術面資料，不可輸出 Bullish 結論
* 若 Freight 是 inferred_from_search，可以輸出 Bullish，但 confidence 必須低於 exact data
* 若資料 stale，Data Coverage 降低
