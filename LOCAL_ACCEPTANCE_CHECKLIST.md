# 本機測試驗收清單

目前階段目標：先完成本機可用、可重跑測試、可手動驗收。雲端部署先保留架構，後續再討論。

## 一鍵測試

雙擊：

```text
RUN_LOCAL_TESTS.bat
```

通過條件：

```text
All local tests passed.
```

這會檢查：

- Python compile
- 單元測試
- 首頁可載入
- 潛力股 API 可回應
- 股票有中文名
- 最多持股與換股候選欄位存在
- 盤前盤中盤後紀錄 API 可回應
- 績效回朔 API 可回應
- 歷史回放回測 API 可回應
- 手續費、交易稅、滑價、持倉延續、benchmark 比較欄位存在
- 績效回朔只統計實際模擬買進，不統計參考掃描或未買進候選
- 歷史回放使用前一交易日訊號、下一交易日開盤模擬成交
- OpenAI 深度解讀開關存在，預設關閉
- 正式持倉延續帳本 API 可回應
- 停損、停利、賣出分數、換股分差、最短持有天數、策略版本欄位存在

## 手動啟動

雙擊：

```text
啟動AI平台.bat
```

打開：

```text
http://127.0.0.1:8011
```

如果看到舊介面，通常代表你開到 `8010` 舊服務，請改開 `8011`。

## 手動功能驗收

### 1. 頁面

應看到：

- 潛力股模擬操作工具
- 搜尋範圍
- 股票池
- 只抓潛力股參考分析
- 盤前進行分析選股
- 盤中執行模擬交易
- 盤後結算今日結果
- 最多持股，預設 5
- 單股上限
- 每日狀況表
- 案件列表
- 下載 Markdown
- 複製報告
- 手續費率、交易稅率、滑價、Benchmark
- 使用 AI 深度解讀
- 賣出門檻，預設 50
- 風險/報酬偏好，預設「中風險 / 中報酬」
- 投資週期，預設「中長線（約 3 個月）」
- 停損，預設 8%
- 停利，預設 20%
- 換股分差，預設 10 分
- 最短持有天數，預設 3 天
- 策略版本，預設 `potential-v1`

### 2. 盤前計畫

操作：

1. 報告類型選「盤前計畫」。
2. 搜尋範圍選「半導體股」。
3. 最多持股維持 5。
4. 按「產生潛力股報告」。

應看到：

- 股票代號加中文名，例如 `2330.TW 台積電`
- 潛力股排行
- 模擬操作顯示「預計買進」
- 盤前不新增正式持股帳本
- 換股候選
- 分析報告

### 2-1. 盤中模擬交易

操作：

1. 先照「盤前計畫」產生一次今日計畫。
2. 按「盤中執行模擬交易」。

應看到：

- 報告標題為「盤中模擬交易」。
- 系統只會針對同一天盤前 `PLAN_BUY` 模擬成交。
- 買進價格使用當下股價加滑價。
- 正式帳本出現 `BUY`、`holdings`、`total_value`。
- 同一天再次按盤中執行，只會讀既有不可變紀錄，不會重複買進。

### 2-2. 盤中風險參考

操作：

1. `13:00` 左右按「只抓潛力股參考分析」。

應看到：

- 只更新畫面與報告。
- 不寫每日紀錄。
- 不新增成交。
- 不改正式持倉帳本。

### 3. 盤後結算

操作：

1. 先照「盤前計畫」產生一次今日計畫。
2. 再照「盤中模擬交易」執行一次。
3. 按「盤後結算今日結果」。
4. 打開 `http://127.0.0.1:8011/api/potential-stocks/ledger`。

應看到：

- 同一天盤前、盤中與盤後被配對
- 盤前買進候選
- 盤中依盤前計畫，用當下股價加滑價模擬成交
- 盤後只更新收盤估值與檢討，不回頭新增買進
- 正式帳本保留 `BUY`、`holdings`、`total_value`
- 盤後分數或狀態
- 價格變化或待補資料

### 4. 持股上限

操作：

1. 最多持股改成 2。
2. 買進門檻可暫時調低，例如 40。
3. 按「產生潛力股報告」。

應看到：

- 模擬持股最多 2 檔
- 其餘達標股票進入「換股候選」

### 5. 績效回朔

操作：

1. 至少產生兩次不同時間的報告。
2. 按「查看績效回朔」。

應看到：

- 累積報告數
- 累積買進訊號
- 已驗證訊號
- 待驗證訊號
- 勝率與平均報酬
- 只納入實際模擬 BUY 交易，不把盤中觀察或換股候選誤算為績效訊號
- 候選股命中率與實際交易績效分開顯示
- 實際帳戶淨值、累積報酬、benchmark 報酬與超額報酬欄位存在

### 5-1. 正式持倉帳本

預設標準：

- 買進門檻：`70`
- 觀察門檻：`55`
- 賣出門檻：`50`
- 停損：`8%`
- 停利：`20%`
- 換股分差：`10` 分
- 最短持有天數：`3` 天
- 最多持股：`5`
- 單股上限：`20%`
- 手續費率：`0.1425%`
- 交易稅率：`0.3%`
- 滑價：`5 bps`
- Benchmark：`0050.TW`
- 策略版本：`potential-v1`
- 風險/報酬偏好：`中風險 / 中報酬`
- 投資週期：`中長線（約 3 個月）`

偏好影響：

- 短線：提高技術面與新聞面權重。
- 中長線：維持技術、基本面、籌碼、新聞的平衡。
- 長線/長期：提高基本面與資料品質權重。
- 高風險/高報酬：較重視技術動能與題材，容忍較高波動。
- 低風險/穩健報酬：較重視資料品質與基本面，資料不足會扣分較重。

手動驗證：

1. 報告類型選「盤前計畫」。
2. 買進門檻可先調低到 `40`，讓本機較容易產生計畫。
3. 按「產生潛力股報告」，確認模擬操作顯示「預計買進」。
4. 報告類型改成「盤後檢討」。
5. 按「盤中執行模擬交易」，系統會依盤前計畫與當下股價模擬成交。
6. 按「盤後結算今日結果」，系統只做結算檢討。
7. 用瀏覽器打開：

```text
http://127.0.0.1:8011/api/potential-stocks/ledger
```

應看到：

- `cash`
- `holdings`
- `trades`
- `total_value`
- `strategy_version`
- `settings`
- `candidates`
- `benchmark`

再按「查看績效回朔」，應看到：

- 實際帳戶最新淨值
- 實際帳戶累積報酬
- 候選股命中率
- 實際交易勝率

### 6. 歷史回放回測

操作：

1. 搜尋範圍選「半導體股」。
2. 模擬資金維持 `1000000`。
3. 最多持股維持 `5`，或改成 `2` 測試持股限制。
4. 手續費率、交易稅率、滑價、Benchmark 可先使用預設值。
5. 按「執行歷史回放回測」。

應看到：

- 期末資產
- 策略報酬
- Benchmark 報酬
- 超額報酬
- 最大回撤
- 交易次數
- 手續費、交易稅與滑價成本
- 期末持股與最近交易
- 最近交易應包含訊號日期與成交日期，避免用同一天收盤訊號同一天成交

### 7. AI 深度解讀

操作：

1. 先保持「使用 AI 深度解讀」未勾選。
2. 按「產生潛力股報告」。
3. 再勾選「使用 AI 深度解讀」。
4. 再按一次「產生潛力股報告」。

應看到：

- 未勾選時不會呼叫 OpenAI，報告維持規則引擎結果。
- 勾選後若 OpenAI key 正常，報告下方會多出「AI 深度解讀」。
- 若 OpenAI 失敗，報告仍會產生，並在資料限制中標示 OpenAI 未完成。
