# 潛力股工具完整優化審查

更新日：2026-06-07

## 目前狀態

這套工具已經具備雛形：

- 可用面板設定股票池、資金、風險、持股上限、交易成本、滑價、benchmark。
- 可分盤前、盤中、盤後執行，並限制順序。
- 可把設定、報告、帳本、案件資料存到 Supabase。
- 可用 cron-job.org 自動觸發。
- 可保留支線與帳本，觀察模擬績效。

目前最大問題不在架構，而在「資訊品質、訊號設計、策略驗證、介面可讀性」還不夠強。

## 優先改善 1：資訊來源補強

目前訊息偏基本資料與一般新聞，容易後知後覺。建議新增更早期、更可量化的訊號：

1. 月營收動能
   - YoY、MoM、近 3 個月累計成長。
   - 營收創高、連續加速、轉正。
   - 與股價反應比對：營收好但股價尚未反應，列為潛在機會。

2. 美國科技領先訊號
   - NVDA、AMD、AVGO、TSM ADR、ASML、AMAT、LRCX、MU、SMH、SOXX。
   - 前一晚漲跌、成交量、突破、族群廣度。
   - 對台股半導體、AI 供應鏈給加權分數。

3. 供應鏈與族群擴散
   - 不只看單股新聞，而是看同族群是否同步轉強。
   - 例如 IC 設計、封測、伺服器、散熱、PCB、電源、設備。

4. 籌碼變化
   - 外資、投信、自營商連買。
   - 融資融券變化。
   - 成交量放大但價格未過熱。
   - 大戶持股集中度若資料可取得，列入中長線權重。

5. 技術結構
   - 多週整理後突破。
   - 相對強度勝過 0050、電子指數、半導體族群。
   - 量價背離、均線多頭排列、波動收斂後放量。

6. 預期差訊號
   - 市場已知道的新聞權重降低。
   - 早於新聞的訊號權重提高，例如營收加速、族群輪動、供應鏈同步、股價相對強度。

## 優先改善 2：評分模型重構

目前分數可以用，但應改成更透明的多層分數：

| 面向 | 建議權重 | 說明 |
| --- | ---: | --- |
| 動能/技術面 | 25% | 趨勢、突破、相對強度、量價 |
| 營收/基本面 | 25% | 月營收、毛利、EPS、估值合理性 |
| 籌碼面 | 20% | 三大法人、融資券、成交結構 |
| 領先因子 | 15% | 美股科技、同族群擴散、供應鏈 |
| 風險控制 | 10% | 波動、過熱、新聞風險、追高風險 |
| 資料品質 | 5% | 資料是否完整、是否延遲 |

每檔股票都應顯示：

- 原始數據。
- 分數。
- 加分理由。
- 扣分理由。
- 是否可交易。
- 若不可交易，是因為分數不足、價格太高、資金不足、或已達持股上限。

## 優先改善 3：策略驗證

為了證明這套 tool 值得跟，需要把「候選股命中率」和「實際交易績效」分開：

- 候選股命中率：被選出後 5、10、20、60 日是否贏 benchmark。
- 實際交易績效：真的依照模擬規則買賣後的報酬。
- 錯過分析：高分但沒買的股票後來是否大漲。
- 誤判分析：買入後跌破停損或輸 benchmark 的共同原因。
- 策略版本比較：`potential-v1`、`potential-v2` 的績效不要混在一起。

## 優先改善 4：交易規則明確化

目前已經有買進、停損、停利、換股分差，但還要更明確：

- 盤前只產生計畫，不成交。
- 盤中依當下價或開盤價加滑價回填成交。
- 未達買進門檻時，可以明確產生「今日不交易」。
- 持股不足最短天數時，除非停損，不輕易賣。
- 若資金不足，要顯示「原本想買但資金不足」。
- 換股要比較新標的分數與舊持股分數，而不是只看新標的高分。

## 優先改善 5：介面重整

目前功能很多，但資訊密度和視覺層級還需要整理：

- 第一區：目前支線、資金、持股、今日狀態。
- 第二區：三個主要操作按鈕。
- 第三區：設定，預設收合。
- 第四區：今日盤前/盤中/盤後，可收合。
- 第五區：持股損益表。
- 第六區：候選股排行與分數拆解。
- 第七區：歷史績效、策略檢討。

重點是讓每天使用時只看「今天該不該做、做什麼、為什麼、結果如何」。

## 建議開發順序

1. 修 UI 可用性與資料同步問題。
2. 重整評分輸出，讓每個分數都有原始證據。
3. 補強月營收、族群強度、美股領先、相對強度。
4. 建立候選股追蹤績效表。
5. 建立策略版本比較。
6. 改版介面視覺與資訊層級。
7. 再加入更進階的 AI 解讀與自我檢討。

## 下一步建議

下一階段先做 `potential-v2`：

- 不推翻現有架構。
- 保留 `potential-v1` 的歷史績效。
- 新增更好的訊號欄位與評分理由。
- 讓每日報告同時顯示 `v1` 與 `v2` 差異，至少跑一段時間比較。

## 專業工具化路線

專業版不應只是更好看的表單，而應該變成「研究台 + 模擬帳戶 + 策略審查」：

1. 工作台首頁
   - 今日狀態、帳戶淨值、現金、持股、市場結論。
   - 盤前、盤中、盤後三個主要流程保持最醒目。
   - 設定放側欄或收合，不干擾每天查看。

2. 證據鏈
   - 每檔股票要有原始數據表：營收、法人、技術、相對強度、族群強度。
   - 每個分數都要有「加分來源」和「扣分來源」。
   - 新聞只列為事件風險，不作為主要領先訊號。

3. 策略檢討
   - 每日自動記錄：今天為何買、為何不買、為何續抱、為何賣。
   - 每週/每月自動檢討：命中率、勝率、平均貢獻、最大虧損、輸 benchmark 原因。
   - 錯過的候選股也要追蹤，避免只看已交易結果。

4. 版本管理
   - `potential-v1` 維持歷史。
   - `potential-v2` 加入月營收、族群擴散、美股領先、相對強度。
   - 同一期間可比較不同策略版本表現。
